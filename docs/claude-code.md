# Claude Code 設定ファイルガイド

このプロジェクトで使用しているClaude Code関連ファイルの説明。

## ファイル一覧

| ファイル | Git管理 | 説明 |
|---|---|---|
| `CLAUDE.md` | する | チーム共有の設定。プロジェクト概要・規約・ブランチ戦略など、Claude Codeに理解させたい共通情報を書く |
| `CLAUDE.local.md` | しない | 個人用の設定。自分のローカル環境固有のパスやコマンド、個人的な作業指示を書く |
| `.claudeignore` | する | Claude Codeに読ませたくないファイルを指定。キャッシュや秘密情報など無関係なファイルを除外してコンテキストを節約する |

## 各ファイルに書くこと

### CLAUDE.md（チーム共有）

Claude Codeが作業する際に常に参照する情報。チーム全員に共通するもの。

- プロジェクト概要・目的
- 技術スタック
- ディレクトリ構成のルール
- コーディング規約（命名規則、言語、テストの書き方など）
- ブランチ戦略
- 開発環境のセットアップ手順

### CLAUDE.local.md（個人用）

自分だけのClaude Codeに伝えたい情報。`.gitignore`に入れてコミットしない。

- ローカル環境のパス・コマンド
- 個人的な作業の好み（出力の言語、コードスタイルなど）
- 今取り組んでいるタスクの補足情報

### .claudeignore（除外設定）

`.gitignore` と同じ書式。Claude Codeのコンテキストに含めたくないファイルを指定する。

- キャッシュ・ビルド成果物（`__pycache__/`, `.aws-sam/`）
- 秘密情報（`.env.local`）
- 大きなバイナリやログファイル

## セットアップ

1. リポジトリをクローン
2. `CLAUDE.local.md` を作成（テンプレートは `.gitignore` 内のためリポジトリに含まれない）
3. コンテナ内で `claude login` を実行してSSO認証
4. `claude` コマンドで開発開始
